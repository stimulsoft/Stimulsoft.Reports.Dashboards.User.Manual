## Schedule

The Schedule point in the File menu contains a list of created Schedulers in the report designer and the command of a new element Scheduler creation. This element allows you to make definite actions with a report by schedule. For example, you can automate the process of report creation, export it to a definite file type and save the result to the local storage or send by email.

All schedulers are located in the Schedule panel point, as a list. To add a new scheduler, you should click the New Scheduler button. When creating a scheduler you should define its type. Depending on a type, schedule settings may vary.
* The Scheduler Once will be run once at a specified date and time, unless otherwise specified by additional parameters;

* The Scheduler Hourly will be run every hour at certain minutes, unless otherwise specified by additional parameters;
* The Scheduler Daily  will be run every day at certain time, unless otherwise specified by additional parameters;
* The Scheduler Weekly will be run on a certain day of the week and time, unless otherwise specified by additional parameters;

* The Scheduler Weekly will be run on a certain day of the week and time, unless otherwise specified by additional parameters.

> **Information**
>
> Any scheduler can be run forcibly. To do that you should select it in the list of schedulers and click the Run button.

More Options

Apart from basic parameters, each scheduler contains additional parameters, which are located on a separate panel. To open this panel, you should click the More Options button.


![](../../images/topics/Reports_Designer.File_Menu.Scheduler_13.png)


![](../../images/img_1.png) The Repeat task every parameter allows you to define the interval of additional run of the current scheduler. The for a duration of parameter allows you to define the interval during which additional runs of the scheduler will be occurred.

![](../../images/img_2.png) The Stop all running tasks at the end of repetition duration parameter allows you to stop the scheduler after the completion of the repetition cycle.

![](../../images/img_3.png) The Delay task for up to parameter allows you to define the delay interval of run the current scheduler.

![](../../images/img_4.png) The Stop tasks if it runs longer than parameter allows you to define a time interval after which the scheduler will be stopped if its tasks are not completed.

![](../../images/img_5.png) The Expire parameter allows you to define date and time when the scheduler is stopped.

Scheduler actions

Each scheduler makes definite actions. By type, all actions can be divided into:
* The Run Report, i.e rendering and exporting a report or a dashboard to a definite document;
* The Send Email, i.e sending a report file or a document to a definite list of people.


Max number of actions in the scheduler is limited up to 15 tasks. You can control the Scheduler tasks on the Actions tab in its editor.


![](../../images/topics/Reports_Designer.File_Menu.Scheduler_14.png)


![](../../images/img_1.png) The list of all scheduler actions.

![](../../images/img_2.png) You should specify a report for the current action in the Report field. Also, if the report contains the parameters, which require user`s request, they can be set using the Parameters special control.

![](../../images/img_3.png) You should define a local place of saving a finished document in the Destination field.

![](../../images/img_4.png) The Result Type parameter allows you to define a type of a file, in which a report or a dashboard will be exported. Please take note that the list of available file types differs for a report or a dashboard. If there is a report or a dashboard in a template, the list of file types will be as well as for the template only with a dashboard.

![](../../images/img_5.png) The Result Name parameter allows you to create a name formation template for the finished document.

![](../../images/img_6.png) The Owerwrite If Exists parameter allows you to define either the finished document will be overwritten every time when the scheduler is run or a copy of the file will be saved for each exporting.

In case of creating the Send Email action, you should fill some fields, too.


![](../../images/topics/Reports_Designer.File_Menu.Scheduler_15.png)


![](../../images/img_1.png) The list of all scheduler actions.

![](../../images/img_2.png) The emails of the receivers, who will get a message when the scheduler is run are specified in the To field.

![](../../images/img_3.png) The theme of the email is specified in the Subject field.

![](../../images/img_4.png) You can specify the text of your email in the Message field.

![](../../images/img_5.png) You should choose exported reports using the Run Report actions, which will be attached to the current email. Please take note that you can`t attach other files to the email apart from the result of the Run Report the result of the action.


> **Information**
>
> Please take note that you should define SMPT settings for the Send Email action. These settings are defined in the Options menu of the report designer on the Scheduler tab.
